Сұрақтарға кішігірім жауаптар:

1.API (Application Programming Interface) - бұл әртүрлі бағдарламалардың өзара әрекеттесуіне мүмкіндік беретін ережелер мен хаттамалар жиынтығы. API-дің негізгі мақсаты-бір бағдарламаның функционалдығына немесе деректеріне басқа бағдарламаға қол жетімділікті қамтамасыз ету. Бұл әзірлеушілерге әртүрлі қызметтер мен қосымшаларды біріктіруге мүмкіндік береді.

2.Http (Hypertext Transfer Protocol) - бұл Интернет желісіндегі деректерді беру протоколы. Ол сұраныс-жауап принципі бойынша жұмыс істейді және веб-беттер сияқты гипермәтіндік құжаттарды беру үшін қолданылады. HTTP клиент пен сервердің бір-бірімен қалай әрекеттесетінін анықтайды.

3.HTTP сұрауларының түрлері:

GET: серверден деректерді сұрайды.
POST: өңдеу үшін деректерді серверге жібереді.
PUT: сервердегі деректерді жаңартады.
Жою: сервердегі деректерді жояды.


4.Кейбір http күй кодтары:

200 жарайды: сәтті сұрау.
201 жасалған: ресурс сәтті құрылды.
400 нашар сұраныс: клиенттің сұрауындағы қате.
401 Unauthorized: авторизация қажет.
404 қор жоқ: сұралған ресурс табылмады.
500 internal server Error: ішкі сервер қатесі.

5.Деректер HTTP сұраулары мен жауаптарында JSON, XML, HTML және т.б. сияқты әртүрлі форматтарда берілуі мүмкін. Деректер мысалдары мәтін, суреттер, бейнелер және басқа медиа файлдар болуы мүмкін.

6.REST (Representational State Transfer) - бұл желілік қосымшаларды жобалауға арналған архитектуралық стиль. REST-тің негізгі принциптері әр ресурс үшін бірегей URI-ді қолдануды қамтиды, CRUD ресурстардағы операциялар (Create, Read, Update, Delete) және ресурстармен өзара әрекеттесу үшін HTTP әдістерін қолдану.

7.JSON (JavaScript Object Notation) - бұл деректерді бөлісудің жеңіл форматы. Бұл адам сияқты оңай оқылатын және бағдарламалармен оңай парсимонияланатын мәтіндік формат. JSON көбінесе желі және API арқылы деректерді тасымалдау үшін қолданылады.

8.Веб-әзірлеуде API-мен жұмыс істеу үшін axios, fetch, jQuery сияқты кітапханалар мен құралдар қолданылады.ajax және Express (серверлік API құру үшін) және Swagger (API құжаттамасы үшін) сияқты құрылымдар.

9.API - мен жұмыс істеу кезіндегі қауіпсіздік деректерді шифрлау үшін HTTPS пайдалануды, пайдаланушылардың аутентификациясы мен авторизациясын, ресурстарға қол жеткізуді шектеуді, қателерді өңдеуді және деректерді тексеруді және т.б. қамтиды. Қауіпсіздік API-ді зиянкестерден қорғауда маңызды рөл атқарады.
